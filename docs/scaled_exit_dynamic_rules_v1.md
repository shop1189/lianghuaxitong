# 分批止盈止损 + 动态价位 — 设计说明 v1

本文档描述如何在**不替代交易所撮合**的前提下，为本地 memos / 模拟记账增加「分批平仓」与「随行情重算 SL/TP」的规则层；实现可分阶段，避免一次性推翻现有 `first_exit_tick` 整笔平仓模型。

---

## 1. 现状（基线）

| 项目 | 当前行为 |
|------|----------|
| 平仓判定 | `utils/trade_exit_rules.first_exit_tick`：单点价 `price` 对 **整笔** 仓位判 SL→TP3→TP2→TP1，**先命中先全平** |
| 盈亏% | 按整笔在该档价位一次性结算 |
| 文案 | 决策页「TP1 平 50%」等为**人工跟单说明**，代码未执行分批 |

---

## 2. 设计目标

1. **分批**：例如 TP1 平 **50%**、TP2 平 **30%**、剩余 **20%** 在 TP3 或跟踪离场（比例可配置）。
2. **动态价位**：在每次行情快照（如决策页/后台 tick）用 **波动、结构、状态** 更新 SL/TP 绝对价，而不是入场时写死到平仓。
3. **可审计**：`trade_memory` 仍能统计胜率、毛/净，可追溯每一「腿」。
4. **可开关**：默认关闭或仅主观察池试验，避免影响现有整笔逻辑。

---

## 3. 仓位状态机（逻辑层）

对每一笔**未完全平仓**的持仓，维护：

| 字段 | 含义 |
|------|------|
| `direction` | 做多 / 做空 |
| `entry0` | 初始入场价（不变，用于展示） |
| `remaining_weight` | 剩余仓位权重 `0~1`（初值 1.0） |
| `vwap_partial` | 已平部分的加权均价（可选，用于展示「已兑现」） |
| `sl / tp1 / tp2 / tp3` | **当前生效**的止损与三档止盈（可每 tick 重算） |
| `tp1_ratio / tp2_ratio / tp3_ratio` | 例如 `0.5 / 0.3 / 0.2`，和为 1 或与 runner 规则一致 |
| `stage` | `open` → `after_tp1` → `after_tp2` → `flat` |
| `regime_snapshot` | 可选：上次动态调价时的 Markov/ATR 指纹，用于防抖 |

**单次 tick 决策顺序（建议）：**

1. 用**当前标记价** `mark` 与**当前档位**判是否触发 **SL**（对**剩余仓位**仍适用；若 SL 触发，**一次性平掉剩余**，记一条 `close_reason: SL`）。
2. 否则按 **TP1 → TP2 → TP3**（或你规定的顺序）检查是否触及；**仅对当前阶段应平的那一档**做部分平仓：
   - 触 TP1：平仓 `remaining_weight * tp1_ratio`，`remaining_weight` 减少；记录一条**分笔**或子记录；`stage` 前进。
3. 若本 tick 未触发任何档，`remaining_weight > 0`，则进入 **动态调价**（下一节）。

> 说明：与现有「SL 优先于 TP3」的**整笔**顺序不同，分批模型里通常 **SL 仍优先**（保护剩余仓位）；TP 各档按阶段依次兑现，避免一次穿过多档时重复计量——需在实现里约定「同一根 K 线内最多处理一档」或「按路径模拟」，见 §6。

---

## 4. 动态止盈止损（随实时行情）

### 4.1 输入（每标的、每快照可用）

- 最新价 `last`、1m/5m **ATR%** 或真实波动幅度  
- 决策快照已有字段：`consistency_score`、Markov/模板态、`experiment_kronos_regime` 等（择一或组合）  
- 可选：布林带宽、近 N 根 K 线高低

### 4.2 调价原则（示例，均可做成环境变量）

| 场景 | SL | TP 带 |
|------|-----|--------|
| 波动放大（ATR% 高于阈值） | 与入场距离**略放宽**（防噪声扫损） | 各 TP 间距**略放大**（保留 RR 比下限） |
| 强趋势（得分与趋势一致） | 可启用**跟踪止损**（仅上移/下移一侧） | TP2/TP3 可**向有利方向递进** |
| 震荡 / chop | SL **收紧**或要求更大 RR 才开仓 | TP1 **收紧**、鼓励早兑现 |
| 调价频率 | 每 **Δt 秒**或「价位变化超过 ε%」才写盘，避免每个 tick 抖动 |

### 4.3 约束

- 任意时刻：`做多` 须 `sl < entry < tp1 < tp2 < tp3`（或与你符号约定一致）；`做空` 对称。  
- 单次调价幅度设 **上下限**（如相对入场 ±0.05%～0.5% 以外需二次确认）。

## 5. 数据落库（`trade_memory` 扩展建议）

**方案 A（推荐演进）：** 父订单 + 子成交  

- 父：`order_id`, `symbol`, `entry0`, `direction`, `virtual_signal`, `status`  
- 子：`parent_id`, `leg`: `partial_sl` / `tp1_partial` / …, `fill_px`, `fill_weight`, `profit_contribution`, `ts`  

**方案 B（最小改动）：** 仍用扁平数组，增加字段  

- `partial_group_id`：同一开仓衍生的多条共用一个 id  
- `leg_index`: 0=开仓, 1=TP1 腿, 2=TP2 腿…  
- 汇总胜率时对 `partial_group_id` **去重**或按**最后一腿**计一笔「完整交易」

统计区块需同步约定：**毛盈亏**可按腿累加，或按组汇总为一条「等效整笔」。

---

## 6. 与现有 `first_exit_tick` 的关系

- **保留**现有函数作为「整笔模式」回归基线。  
- 新增模块（例如 `utils/scaled_exit_engine.py`）：  
  - `on_tick(state, mark_price, km_snapshot) -> List[FillEvent]`  
  - 单元测试：单档、穿多档、SL 与 TP 同 tick 等边界。

---

## 7. 实现阶段建议

| 阶段 | 内容 |
|------|------|
| **P0** | 仅文档 + 配置项占位（比例、开关、ATR 阈值） |
| **P1** | 内存态分批 + 仍写**多条**扁平 `trade_memory`（方案 B），动态价位**每日一次**或**入场时一次** |
| **P2** | 每决策刷新动态 SL/TP + 防抖；页面展示「当前生效档位」 |
| **P3** | 父/子结构（方案 A）+ 回测对齐 |

---

## 8. 风险与说明

- 本地模拟**不等于**交易所滑点、手续费分层；净盈亏仍建议用现有 **0.16%** 双边近似。  
- 分批后「胜率」定义需统一：按**组**算一笔还是按**腿**算，须在统计模块显式选择。  

---

## 9. 与决策页文案对齐

当 P1+ 落地后，可将决策块文案从「平 50%」改为「**模拟层**：TP1 平 50%（若开启分批模式）」，避免与整笔模式混淆。

## 10. 已实现（环境变量，默认关）

| 变量 | 含义 |
|------|------|
| `LONGXIA_DYNAMIC_LEVELS` | `1`/`true`/`yes`/`on`：主观察池虚拟单开仓与决策快照中的 SL/TP1/TP2/TP3 按 `utils/dynamic_levels.widen_levels_from_closes` 随近期收盘波动微调；未设或 `0` 则与改前一致。 |
| `LONGXIA_SCALED_EXIT` | `1`/`true`/`yes`/`on`：主观察池虚拟单启用分批（`utils/scaled_exit_rules.try_scaled_virtual_close`）；新开仓写入 `scaled_mode` 等字段；未设或 `0` 则仍用 `first_exit_tick` 整笔平仓（历史带 `scaled_mode` 的记录也会整笔平）。 |
| `LONGXIA_SCALED_W1` / `W2` / `W3` | 可选；三档占原始名义比例，默认 `0.5` / `0.3` / `0.2`，可和不为 1（会归一化）。 |
